方向选择：

我会选择以运营为主，辅以技术的方向，主要是为了在活动的推广中清晰明白自己推广的事物原理与构建逻辑，以防止在一无所知的情况下就进行建设。主要就意图服务于在运营的同时可以解答部分详细的技术问题。本周最小产出目标定为在技术方面整合好上周所做的留言板全功能，添加后端部分并优化整体设计。在Week3中我可以尝试担当项目宣传和一部分的技术参与，帮助完成项目。

AI-assisted Dev Plan（Tec）：

我准备制作一个最小的 Monad 链上留言板。用户连接 钱包 后，可以读取当前链上留言，并通过一笔交易更新留言。留言不会保存在普通数据库或本地缓存中，而是写入 Monad Testnet 智能合约。如果我拥有更充足的时间，我会考虑将它升级为一个聊天室而不只是留言板。

这个Dapp主要面向第一次体验 Web3 DApp 的新手，以及希望了解 Monad 链上 read / write 流程的营员和开发者。用户需要完成的包括连接钱包，输入一句留言，点击更新按钮，在钱包中确认交易，等待交易完成后看到新的链上留言。如果后续改为聊天室，用户可以完成还有自定义个人的展示信息。

策划案（Ops）：

活动主题

Web3 不只是炒币：普通互联网用户的第一堂 Web3 入门课

本活动计划采用线上 Space 或 AMA 的形式，规模控制在 30–100 人。内容面向没有系统接触过区块链和 Web3 的普通互联网用户，以浅显的方式介绍 Web3 的基本概念、实际使用场景和安全注意事项。

本次活动希望帮助参与者建立一个相对准确的 Web3 初步认知，而不是单纯宣传代币或投资机会。在活动结束后，希望参与者能够理解

  • 区分 Web3、区块链和炒币；
  • 理解钱包、地址、智能合约和区块浏览器的基本作用；
  • 知道普通互联网产品与链上产品的主要区别；
  • 了解 Web3 中内容、社区、研究、产品和技术等不同参与方向；
  • 掌握最基础的安全原则，例如不泄露助记词和私钥。

主要会尝试面向对 Web3 有兴趣，但不知道从哪里开始的普通互联网用户；学生、内容创作者、社区运营和互联网产品从业者；想进入 Web3 学习或实习，但没有编程基础的新人；认为Web3 只是炒币、诈骗或高门槛技术等人群。尝试解决Web3 是否等于炒币？钱包、地址和智能合约到底是什么？普通人能以什么方式参与？怎样安全地完成第一次体验？四个核心问题

因为Web3 的传播经常被投资、币价和技术术语主导，同时新人了解该行业面临的信息风险较高，而小型线上活动可以增强双方互动，适合新人提问形成有效交流氛围，故因此值得开展。
